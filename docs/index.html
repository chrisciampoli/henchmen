Why Henchmen

Stripe has Minions. You have Henchmen.

An open-source answer to the agent factories that big companies build for themselves.

I spent six months watching Cursor and Claude Code eat my attention. I’d ask for something, then sit there watching the agent type. It felt like pair programming with a very fast, very literal intern who needed a driver in the seat at all times.

Then I read Stripe’s post about Minions — their internal agent factory that ships 1,000+ PRs per week from Slack messages, entirely unattended. And Alex Op’s Software Factory essay arguing that the next team model is specs-in, PRs-out with AI handling the implementation.

I wanted that. I wanted to fire a task from Slack, walk into a meeting, and come back to a PR in my review queue — reviewed the same way I’d review a teammate’s work. Stripe built it internally. Nothing existed as open source.

So I built Henchmen: an agent factory that runs on your laptop, your cloud, or both. Tasks in, PRs out. Fail-closed CI gates. BYO models. The villain theme came second.

How It Works

Tasks in. Pull requests out.

You write the task. Henchmen writes the PR. You review it. That’s the whole loop.

Step 01

Submit a Task

Send a task from any interface. Dispatch normalizes it into a structured model and publishes it to the queue — no manual formatting required.

Slack GitHub Jira CLI
Step 02

Agents Execute in Isolation

Mastermind selects a Scheme, spins up an ephemeral Operative container, and arms it with the full Arsenal of coding tools. The agent works, then disappears.

Any LLM (GPT / Claude / Gemini / Ollama) Ephemeral containers RAG-grounded context
Step 03

A PR Lands in Your Review Queue

Mastermind verifies the Operative’s work, runs lint and tests through Forge, and opens a pull request on your repo. If CI breaks, the PR never sees daylight. No exceptions.

GitHub PR Fail-closed CI Human review gate

Components

Seven villain-themed components

Each component has one job. No bleed, no bloat — clean separation enforced by Pub/Sub boundaries and Pydantic contracts.

Mastermind
The Boss

Sits at the top of the org chart. Reads every task, picks the right Scheme, dispatches Operatives, and fires the ones that botch the job. Fail-closed CI gates — no broken PR reaches your review queue. Ever.

Dispatch
The Fixer

Receives jobs from Slack, GitHub, Jira, or CLI. Normalizes every request into a single Task contract and hands it off. Doesn’t think. Just routes. One job, done clean.

Operative
Disposable Agent

Spawns in an isolated container, executes its assigned Scheme node, leaves no trace. Armed with the Arsenal, briefed by the Dossier, watched by the Mastermind. Works, then disappears.

Arsenal
The Toolkit

What every Operative carries into the field: code_edit, code_intel, git_ops, test_runner, github. Co-located inside the container for zero-latency tool calls.

Forge
Quality Control

Runs CI on every Operative’s branch, manages the merge queue, catches silent failures. If CI breaks, the PR never sees daylight. No exceptions.

Dossier
Intelligence

The intel file. Gathers project rules, performs RAG-grounded semantic search over the codebase, and builds a per-task brief before the Operative ever touches the keyboard. Cached so the next Operative gets it for free.

Schemes
The Blueprints

Declarative DAG workflows committed to your repo. Version-controlled. Reviewable. Customizable. Comes with bugfix_standard, feature_standard, and goal_decomposition. Fork one. Write your own. Assign different models to different nodes.

Quick Start

Local end-to-end in 10 minutes

Python 3.12+, Docker Desktop, and a GitHub token. Submit a task, watch the operative code, get a PR.

bash — step 1: install & configure
# Clone and install
git clone https://github.com/chrisciampoli/henchmen && cd henchmen
pip install -e ".[dev]"
# Configure — set your GitHub token and LLM choice
cp .env.example .env.local
$EDITOR .env.local
# .env.local (minimum required)
GITHUB_TOKEN=ghp_your_token_here
HENCHMEN_LLM_PROVIDER=openai # or: anthropic, local (Ollama)
HENCHMEN_OPENAI_API_KEY=sk-... # for OpenAI
bash — step 2: start the server
henchmen serve
✓ Starting Henchmen in single-process mode (provider=local)
✓ HTTP forwarding configured for 4 topics
✓ Uvicorn running on http://0.0.0.0:8000
# First run auto-builds the operative Docker image (~2 min)
bash — step 3: submit a task and get a PR
# In a new terminal — send a task
curl -X POST http://localhost:8000/dispatch/api/v1/tasks \
-H "Content-Type: application/json" \
-d '{"title": "Create a Python todo app",
"description": "Create todo.py with add, list, remove",
"repo": "your-user/your-repo"}'
✓ Task dispatched: ab8f2df7
Mastermind selects scheme: feature_standard
Operative spawned in Docker → writing code → pushing branch
Verify changes → lint → tests → create PR
✓ PR created: https://github.com/your-user/your-repo/pull/1
Connect Your Tools

Integrate in minutes

Henchmen receives tasks from four sources. Each takes less than 5 minutes to configure.

🐙

GitHub App

Required for the full PR workflow. Henchmen creates branches, pushes commits, and opens pull requests.

  • Create a GitHub App with Contents + PR permissions
  • Install on your target repository
  • Download the private key (.pem file)
HENCHMEN_GITHUB_APP_ID=123456
HENCHMEN_GITHUB_APP_PRIVATE_KEY_SECRET=/path/to/key.pem
HENCHMEN_GITHUB_DEFAULT_REPO=your-org/your-repo
💬

Slack

@mention the bot in any channel to create a task. Thread context is automatically included for richer descriptions.

  • Create a Slack App with Socket Mode enabled
  • Add bot scopes: app_mentions:read, chat:write
  • Subscribe to the app_mention event
HENCHMEN_SLACK_BOT_TOKEN_SECRET=xoxb-...
HENCHMEN_SLACK_APP_TOKEN_SECRET=xapp-...
HENCHMEN_SLACK_SIGNING_SECRET=your-secret
📋

Jira

Automatically pick up Jira issues when they’re created or transitioned. Henchmen reads the ticket and gets to work.

  • Create a webhook pointing to Dispatch
  • Select issue created + updated events
  • Generate an API token for your service account
HENCHMEN_JIRA_BASE_URL=https://your-org.atlassian.net
HENCHMEN_JIRA_API_TOKEN_SECRET=your-token
HENCHMEN_JIRA_PROJECT_KEY=PROJ
⌨

CLI

Submit tasks directly via HTTP. Perfect for scripting, CI pipelines, or quick one-off fixes from the terminal.

  • No setup required — works out of the box
  • POST JSON to the Dispatch endpoint
  • Pipe output from other tools
curl -X POST http://localhost:8000/dispatch/api/v1/tasks \
  -H "Content-Type: application/json" \
  -d '{"title": "...", "repo": "you/your-repo"}'
