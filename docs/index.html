How It Works

Three steps to autonomous pull requests

Henchmen handles everything between “here’s a task” and “here’s a PR” — so you can focus on reviewing, not implementing.

Step 01
📥

Submit a Task

Send a task from any interface. Dispatch normalizes it into a structured model and publishes it to the queue — no manual formatting required.

Slack GitHub Jira CLI
Step 02
🤖

Agents Execute in Isolation

Mastermind selects a Scheme, spins up an ephemeral Operative container, and arms it with the full Arsenal of coding tools. The agent works, then disappears.

Gemini 2.5 Pro Cloud Run Jobs RAG via Pinecone
Step 03
🔀

Get a Human-Reviewable PR

Forge runs CI, verifies lint and tests pass, and opens a pull request. Fail-closed gates ensure nothing reaches human review if checks are broken.

GitHub PR CI Gates Merge Queue

Components

Seven villain-themed components

Each component has one job. No bleed, no bloat — clean separation enforced by Pub/Sub boundaries and Pydantic contracts.

🧠
Mastermind
Cloud Run Service

Orchestrator. Manages task lifecycle via state machine, selects Schemes, dispatches Operatives. Fail-closed CI gates — never creates PRs when checks fail.

🚦
Dispatch
Cloud Run Service

Intake router. Receives tasks from Slack (Socket Mode), Jira, GitHub, CLI. Normalizes everything to the Task model. Publishes to Pub/Sub — nothing more.

🤖
Operative
Cloud Run Job

Coding agent. Bootstraps into an ephemeral environment, executes Scheme nodes, uses Arsenal tools, reports results. Gone when the job ends.

🛠️
Arsenal
Tool Registry

Runs inside Operative. Provides: code_edit, code_intel, github, git_ops, test_runner. Co-located for zero-latency tool calls.

⚒️
Forge
Cloud Run Service

CI and merge queue. Orchestrates CI pipelines, builds PRs, manages the merge queue, and detects silent failures before anything reaches human review.

📂
Dossier
Library

Context builder. Gathers project rules, performs RAG via Pinecone (henchmen-code index), and caches artifacts to GCS for reuse across Operatives.

📋
Schemes
DAG Blueprints

Workflow execution plans. DAG-based blueprints defining node sequences, model assignments, and retry logic: bugfix_standard, feature_standard, goal_decomposition.

Quick Start

Up and running in minutes

Python 3.12+, Docker, and Git. No cloud account required for local development.

bash — step 1: install
# Clone and install
git clone https://github.com/chrisciampoli/henchmen && cd henchmen
pip install -e ".[dev]"
# Configure for local mode
cp .env.example .env.local
# Start all services + Ollama LLM
docker compose up
✓ Dispatch listening on :8001
✓ Mastermind listening on :8002
✓ Forge listening on :8003
✓ Ollama ready (llama3.2)
bash — step 2: submit your first task
# Send a task to Dispatch
curl -X POST http://localhost:8001/tasks \
-H "Content-Type: application/json" \
-d '{"title": "Fix login bug",
"description": "Users cannot log in after password reset",
"source": "cli",
"repo": "your-org/your-repo"}'
✓ Task created: t-8f3a2b1c
Mastermind selects scheme: bugfix_standard
Operative spawned → analyzing code → committing fix
Forge running CI → lint passed → tests passed
✓ Pull request created: #42
bash — alternative: single process (no Docker)
# Run everything in one process
pip install -e "."
henchmen serve
✓ Henchmen running in single-process mode
Provider: local (SQLite + filesystem + in-memory queues)
LLM: Ollama at http://localhost:11434
Health: http://localhost:8000/health
Connect Your Tools

Integrate in minutes

Henchmen receives tasks from four sources. Each takes less than 5 minutes to configure.

🐙

GitHub App

Required for the full PR workflow. Henchmen creates branches, pushes commits, and opens pull requests.

  • Create a GitHub App with Contents + PR permissions
  • Install on your target repository
  • Download the private key (.pem file)
HENCHMEN_GITHUB_APP_ID=123456
HENCHMEN_GITHUB_APP_PRIVATE_KEY_SECRET=/path/to/key.pem
HENCHMEN_GITHUB_DEFAULT_REPO=your-org/your-repo
💬

Slack

@mention the bot in any channel to create a task. Thread context is automatically included for richer descriptions.

  • Create a Slack App with Socket Mode enabled
  • Add bot scopes: app_mentions:read, chat:write
  • Subscribe to the app_mention event
HENCHMEN_SLACK_BOT_TOKEN_SECRET=xoxb-...
HENCHMEN_SLACK_APP_TOKEN_SECRET=xapp-...
HENCHMEN_SLACK_SIGNING_SECRET=your-secret
📋

Jira

Automatically pick up Jira issues when they’re created or transitioned. Henchmen reads the ticket and gets to work.

  • Create a webhook pointing to Dispatch
  • Select issue created + updated events
  • Generate an API token for your service account
HENCHMEN_JIRA_BASE_URL=https://your-org.atlassian.net
HENCHMEN_JIRA_API_TOKEN_SECRET=your-token
HENCHMEN_JIRA_PROJECT_KEY=PROJ
⌨

CLI

Submit tasks directly via HTTP. Perfect for scripting, CI pipelines, or quick one-off fixes from the terminal.

  • No setup required — works out of the box
  • POST JSON to the Dispatch endpoint
  • Pipe output from other tools
curl -X POST http://localhost:8001/tasks \
  -H "Content-Type: application/json" \
  -d '{"title": "...", "source": "cli"}'
